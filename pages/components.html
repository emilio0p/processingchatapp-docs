Descripción del diagrama de componentes


Este proyecto se compone de diferentes partes que trabajan juntas para dar la experiencia que los usuarios ven.

En la parte superior del diagrama tenemos al 'Usuario', que representa a las personas que utilizan nuestra aplicación.

Justo debajo del usuario, encontramos la parte del 'Front (Cliente)', que es la cara visible de la aplicación, lo que los usuarios ven y con lo que interactúan. Aquí utilizamos Angular para crear una interfaz amigable de chats.

A la derecha, está el 'Servidor', que es donde se procesan las solicitudes de los usuarios. Utilizamos Node.js y JavaScript para manejar esta parte y asegurarnos de que todo funcione correctamente.

En la parte inferior, encontramos un componente clave llamado 'API', que actúa como un intermediario entre el front-end, el servidor y la base de datos. Utilizamos FastAPI para esta capa, que nos permite gestionar las solicitudes de manera rápida y eficiente.

A la derecha del todo, tenemos la 'Base de datos', que es donde almacenamos toda la información necesaria para que la aplicación funcione. Utilizamos MySQL para gestionar eficientemente los datos. En resumen, nuestro diagrama muestra cómo todas estas partes se conectan y trabajan juntas para ofrecer una experiencia fluida a los usuarios de nuestra aplicación."
